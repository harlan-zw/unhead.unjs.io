Capo.js Benchmark Test Page

This page tests head tag ordering impact on performance metrics.

About This Test

This benchmark page measures the impact of head tag ordering on web performance metrics including Largest Contentful Paint (LCP), First Contentful Paint (FCP), and Time to Interactive (TTI).

The head tags on this page are arranged in the Capo.js recommended optimal order, prioritizing critical rendering resources and metadata.

Performance Metrics

Key metrics being measured include render-blocking resource load times, parser-yielding behavior, and resource discovery order through the preload scanner.

Each variant of this page contains identical body content and the same set of head tags, differing only in the order of those tags within the head element.

Feature Cards

Resource Hints

Preconnect, prefetch, and dns-prefetch directives help the browser establish connections and fetch resources earlier in the page load process.

Script Loading

Async and defer attributes change when scripts execute relative to HTML parsing, affecting both interactivity and render timing.

Critical CSS

Stylesheet placement in the head determines when the browser can begin rendering content, directly impacting FCP and LCP metrics.

Font Loading

Font preloading and display strategies affect text rendering and layout stability, contributing to CLS and LCP scores.

Meta Tags

Charset and viewport meta tags placed early ensure correct encoding and layout from the start of parsing.

Third-Party Scripts

Analytics and tag manager scripts can block or delay rendering if not properly positioned with async or defer attributes.

Methodology

Each page variant is tested with 50 Lighthouse runs under both Fast 4G and Slow 3G network conditions. Results are aggregated to produce statistically significant comparisons between ordering strategies.

The test uses simulated throttling to ensure consistent and reproducible results across runs. All pages are served from the same local server to eliminate network variability.
