# Design Guidelines — Unhead

> Generated by `nuxt-frontend-design-system`. This file captures the *intent* behind design decisions so future work stays cohesive across conversations, contributors, and skills.

## Aesthetic Direction

- **Theme**: Custom amber/warm (distinct from nuxtseo.com's violet/mauve)
- **Mode**: both (dark primary, light supported)
- **Vibe**: textured technical warmth
- **Influences**: Developer tool sites with intentional craft, analog warmth via noise grain, amber/gold energy. Not clinical, not playful.

## Design Principle

We prioritize **textured depth and warmth** over clean minimalism. The noise grain overlay, warm-tinted neutrals, and amber accents create a distinctive tactile quality that separates this site from generic documentation. Every surface should feel warm, not cold.

### Premium Quality Bar

Premium here means intentional warmth: amber-tinted neutrals that feel cohesive, noise grain that adds analog character, deep dark mode with proper layer separation. Where nuxtseo.com achieves premium through cool translucent glass, unhead achieves it through warm texture and golden light.

## Color

### Palette

```ts
colors: {
  primary: 'amber',
  neutral: 'stone', // overridden to oklch amber-tinted scale (hue 75)
}
```

### Warm-Tinted Neutral Scale (oklch, hue 75)

The entire neutral scale is overridden in `global.css` `@theme` using oklch with hue 75 (warm amber). This is the single biggest differentiator from nuxtseo.com, which uses hue 285 (cool mauve). The chroma is subtle (0.003 to 0.018), creating an ambient warmth without looking yellow.

### Color Roles

| Role | Value | Usage |
|------|-------|-------|
| Primary | `amber` | CTAs, active states, brand accent |
| Neutral | oklch amber-tinted stone (hue 75) | Body text, backgrounds, borders via semantic tokens |
| Green | Custom palette (#00DC82 family) | UnJS ecosystem, success states |
| Blue | `#82AAFF` | Code highlights (Material Theme Palenight), prose link decorations |

### Color Restraint

- **One accent per context.** A section or card gets one tint at most.
- **Mode-specific saturation.** Light mode uses desaturated ends (`-50`, `-100`). Dark mode uses base at `/5` or `/10` opacity.
- **Neutral is the default.** Color is the exception that draws attention.

### Semantic Tokens

Use Nuxt UI shorthands: `text-default`, `text-muted`, `text-dimmed`, `text-toned`, `text-highlighted`, `bg-muted`, `bg-elevated`.

Only use `[var(--ui-*)]` syntax when you need Tailwind opacity modifiers.

**Dark mode overrides** (all mapped to the amber-tinted neutral scale):
- `--ui-bg`: neutral-950 (deep warm black, ~11% lightness)
- `--ui-bg-muted`: neutral-900
- `--ui-bg-elevated`: neutral-800
- `--ui-text`: neutral-300 (soft warm white, 87% lightness)
- `--ui-text-highlighted`: neutral-100
- `--ui-border`: neutral-800

**Light mode overrides**:
- `--ui-text`: oklch(0.370 0.014 75) (warm dark, not pure gray)
- `--ui-text-highlighted`: oklch(0.210 0.012 75)

### 60-30-10 Split

60% amber-tinted neutrals (backgrounds, surfaces), 30% text/borders, 10% amber accent on CTAs and brand moments.

### Contrast & Accessibility

- **Body text contrast**: warm neutral-700 on light, warm neutral-300 on dark, meeting AA standards
- **Dark mode adjustments**: text softened to neutral-300 (not 200) for reading comfort; code accent brighter (#82AAFF)
- **Known risks**: Amber on light backgrounds can be low contrast; primary buttons use gradient with dark text for legibility

## Signature Effects

### Noise Grain Overlay

The fractal noise texture applied via `body::before` is the site's most distinctive visual element. It creates an analog, textured quality across all surfaces.

- **Implementation**: SVG `feTurbulence` (fractalNoise, baseFrequency 0.75, 4 octaves), tiled at 128px
- **Light mode opacity**: 6%
- **Dark mode opacity**: 5.5%
- **z-index**: 10 (above content, pointer-events: none)
- **Rule**: Never remove or reduce this below 3%. It is the design signature.

### Ambient Gradient

The `.gradient` class renders a fixed radial amber glow behind page content:

```css
/* oklch-based amber glow with soft falloff */
background: radial-gradient(50% 50% at 50% 50%, oklch(0.82 0.15 85), oklch(0.75 0.12 75 / 0.6) 30%, transparent 70%);
filter: blur(150px);
opacity: 0.08; /* light mode */
```

Dark mode reduces opacity to 0.07.

## Typography

| Role | Font | Why |
|------|------|-----|
| Body (`--font-sans`) | Nunito Sans | Rounded, friendly sans-serif; readable at small sizes for documentation |
| Display (`--font-display`) | Hubot Sans | GitHub's variable-width display face; applied to all h1-h6 globally via CSS |
| Mono (`--font-mono`) | Fira Code | Ligature-capable monospace for code blocks, familiar to developers |

- **Heading rule**: All `h1`-`h6` elements use `font-family: var(--font-display)` via global CSS. No need to add font classes to headings.
- **Type system**: Fixed rem scale. Documentation UI with predictable hierarchy.
- **OpenType features**: Hubot Sans uses variable stretch (75%-125%) for display flexibility.

## Icons

- **Collection**: Carbon (IBM Carbon Design System)
- **Why**: Clean 2px strokes, comprehensive set covering developer/technical concepts. Supplemented with Noto emoji icons for section headers on the landing page.
- **Color rule**: Monochrome for UI icons. Noto full-color emoji used sparingly for section personality on marketing pages only.

## Component Rules

- Buttons: `defaultVariants` set to solid + primary. CTA uses amber-to-gold gradient with dark text. Secondary actions use ghost + neutral.
- Cards: `UPageCard` with spotlight effect for feature grids. Rounded-xl globally via app.config.
- Radius: Default Nuxt UI radius via `--ui-radius`. Cards use `rounded-xl`.
- Links (prose): Underline with offset, `--ui-text-secondary` (#82AAFF) decoration that thickens on hover.
- Navigation: Reka UI mega menu with glassmorphism viewport (`shadow-xl rounded-md ring-1`). Muted text for inactive items.
- Tabs: Custom compact styling with `leading-8`, no padding on triggers, bottom-aligned.
- Content navigation: Left border indicator (2px primary) for active items, muted text with hover transition for inactive.
- Code blocks: Material Theme Palenight for dark, GitHub Light for light. Custom warm scrollbar styling.

## Spatial & Motion

- **Spacing system**: Tailwind default (4px base grid)
- **Spacing philosophy**: Generous vertical rhythm on landing page (py-10 to py-20 sections). Documentation uses tighter spacing.
- **Transition speed**: CSS transitions for color/opacity. Shiki Magic Move for code animations.
- **Animation style**: `slide-right` (400ms elastic) and `pop` (400ms elastic) keyframe animations defined in global CSS. Used sparingly.
- **Reduced motion**: No explicit prefers-reduced-motion handling yet. Animations are decorative, not blocking.

## Responsive Strategy

- **Approach**: Mobile-first, breakpoints at sm/md/lg/xl/2xl. Landing page uses 6-column grid at xl with sticky sidebar pattern.
- **Input method**: Standard touch targets. No explicit pointer:coarse adjustments.
- **Navigation adaptation**: Hamburger menu on mobile, horizontal navigation menu on desktop.

## Voice & Tone

- **Button labels**: Verb-object pairs ("Get Started", "Install Unhead", "Become a sponsor"). Action-oriented.
- **Error style**: Follows Nuxt UI defaults.
- **Empty states**: Documentation focus means most states have content.

## Avoid

- Pure black backgrounds or text; always use amber-tinted neutrals (hue 75)
- Cool-tinted grays (blue, mauve); the warmth is the identity, breaking it breaks the design
- Ghost buttons for primary CTAs; they disappear on textured backgrounds
- Colorful icons in navigation or UI chrome; color reserved for emoji section headers and semantic states
- System fonts for headings; always use Hubot Sans display font
- Inline styles for colors or spacing; use Tailwind classes and design tokens
- Heavy drop shadows in dark mode; use lighter surfaces for depth instead
- Bounce/elastic easing for UI transitions; reserved only for the decorative slide-right/pop keyframes
- Removing or reducing the noise grain below 3%; it is the design signature
- Translucent glass effects (that's nuxtseo's identity); use warm solid surfaces with the noise grain instead
- Same color value across both modes; always provide mode-specific values
- Alternating bg-muted section backgrounds on landing pages; use `.section-divider` gradient lines instead

## Custom Utilities

| Class/Token | What it does | When to use |
|-------------|-------------|-------------|
| `.gradient` | Fixed position oklch amber glow, blurred, mode-aware opacity | Landing page background atmosphere only |
| `.slide-right` | 400ms elastic slide-in from right with scale | Decorative entrance for feature showcases |
| `.pop` | 400ms elastic scale-up from 0 | Decorative entrance for badges/small elements |
| `--ui-text-secondary: #82AAFF` | Material Palenight blue | Code-related accents, prose link decorations |
| `--container-width: 90rem` | Wide container max | Root-level layout constraint |
| `body::before` noise | SVG fractal noise overlay, mode-aware opacity | Applied globally, never remove |
| `.section-divider` | 1px gradient line, fades from transparent → border color → transparent, max-width 80%/60rem | Between landing page sections instead of bg-muted alternation |

## Design Decisions

- Landing page sections use sticky sidebar layout at xl; icon + heading on left, content on right
- Stats section uses font-light text-6xl for large numbers; intentional lightweight treatment for data
- Sponsor avatars use rounded-full with optional green ring for higher-tier sponsors
- Code examples are the hero content; Magic Move animations are the primary interactive showcase
- Noise grain at 6%/5.5% (light/dark) is intentional: visible enough to feel textured, subtle enough to not interfere with reading
- Dark mode neutral-300 text (not 200) is intentional: softer reading comfort, matching nuxtseo's approach but with warm tint instead of cool
- oklch hue 75 for all neutrals is intentional: creates ambient warmth without looking yellow, distinctly warm vs nuxtseo's hue 285
- Hero `<head>` uses italic font-display with text-primary (amber) instead of cursive font: reinforces the display font system
- Subtitle accents use text-primary (amber) instead of blue: blue is reserved for code highlights and prose link decorations only
- Inline code syntax colors (#6F42C1, #82AAFF, #24292E, #BABED8) in helloUnheadTitle are intentional: match GitHub Light / Material Palenight code themes
- Landing page uses gradient divider lines between sections instead of alternating bg-muted backgrounds: creates a unified surface where the noise texture is consistently visible
