## Key Findings

All primary brand terms are losing clicks despite stable/growing impressions:

**Root cause hypothesis:** The v2 site migration split authority. `/v2` is now getting 25 clicks that previously went to `/`. Also possible CTR drop from AI Overviews eating branded searches.

- `/docs/react/head/guides/get-started/installation` (107 clicks) + `/installation/` (14 clicks)

Combined these represent ~50+ lost clicks from split signals.

## Q1 2026 Action Plan (Mar-Apr)

### ~~P0: Trailing Slash Consolidation~~ RESOLVED

Was a Cloudflare redirect misconfiguration - now fixed. Monitor GSC over next few weeks to confirm trailing slash variants deindex and authority consolidates.

### P0: Title Tag Template Update (Week 1)

**Impact: High | Effort: Low** - CTR improvement across all pages.

Switch from API-reference style titles to solution-oriented format:

**Template:** `[Solution/Action] | [Tool] | Unhead`

| Page | Old Title | New Title |
|------|----------|-----------|
| LocalBusiness schema | `LocalBusiness - Schema.org API` | `LocalBusiness Schema (JSON-LD Examples & Guide) \| Unhead` |
| HowTo schema | `HowTo - Schema.org API` | `HowTo Schema (JSON-LD Examples & Guide) \| Unhead` |
| useHead | `useHead - API` | `useHead - Head Management Composable \| Unhead` |
| useSeoMeta | `useSeoMeta - API` | `useSeoMeta - SEO Meta Tags Composable \| Unhead` |
| React install | `Installation - React` | `React Head Management - Get Started \| Unhead` |
| Overview | `Overview - Get Started` | `Unhead - Universal Head Management for JavaScript` |

Meta descriptions should lead with a factual 120-char summary for AI Overview extraction.

### P0: /v2 Consolidation (Week 1)

- [ ] Add `noindex` or canonical to `/v2` pointing to main docs
- [ ] 25 clicks currently leaking to deprecated v2 pages

### P1: Schema.org Page Revamp (Week 2-4)

**Impact: High | Effort: Medium** - Transform 0.03% CTR into 2-3% = 300-500 new monthly clicks.

Revamp the top 6 schema type pages. Winning pages in SERPs use an "Answer-First" format that AI Overviews cite.

#### Content structure for each schema page:

1. **"What is X Schema?"** intro (2-3 sentences, answers search intent immediately)
2. **Simulated rich result preview** showing what the schema produces in Google
3. **Copy-pasteable JSON-LD example** (the #1 thing searchers want)
4. **Required vs Optional properties table** with 2026-specific notes:
   - LocalBusiness: flag that subtypes (Dentist, ProfessionalService) are preferred over generic
   - All types: highlight `geo`, `hasMap`, `sameAs` as "Critical for AI Visibility"
5. **Unhead composable usage** (the existing API reference content)
6. **Cross-link to schema generator tool**

#### Priority order (by impression volume):
1. [ ] LocalBusiness (~8,000 impressions, pos 8-10)
2. [ ] SoftwareApp (5,200 impressions, pos 8.1)
3. [ ] JobPosting (~3,400 impressions, pos 5.7) + create `/docs/schema-org/guides/recipes/job-posting`
4. [ ] HowTo (~3,200 impressions, pos 8-10)
5. [ ] ItemList (~2,800 impressions, pos 5-8)
6. [ ] Article (~2,500 impressions, pos 7-9)
7. [ ] Organization (~500 impressions, pos 9-10)

#### Example metadata:
- **Title:** `How to Implement LocalBusiness Schema (JSON-LD Examples) | Unhead`
- **Description:** `Master LocalBusiness structured data in 2026. Copy-pasteable JSON-LD examples, mandatory properties for AI visibility, and rich result previews.`

### P2: React Migration Page (Week 3-4)

**Impact: Medium | Effort: Low** - Capitalize on only growing segment.

- [ ] Create dedicated `/docs/react/head/guides/get-started/migrate-from-react-helmet` improvements or a new comparison landing
